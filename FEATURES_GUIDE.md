# 餐廳管理系統 - 功能使用指南

> **開發工具**: 本專案使用 [opencode](https://opencode.ai) + GitHub Copilot 進行 AI 輔助開發

## 🌟 核心功能概覽

本系統提供完整的餐廳營運管理功能，採用現代化的 React 架構，提供直觀易用的操作界面。

## 📊 儀表板功能

### 即時數據顯示
- **今日營收**: 當日所有已完成訂單的總金額
- **今日訂單數**: 當日建立的訂單總數
- **目前用餐人數**: 所有用餐中桌位的總人數

### 快速操作
- 一鍵新增訂單
- 快速查看各功能模組
- 即時數據自動更新

## 🪑 智能桌位管理

### 桌位狀態一覽
系統提供 12 個桌位的視覺化管理界面：
- **綠色 (空桌)**: 可直接點擊開始點餐
- **紅色 (用餐中)**: 點擊查看桌位詳情

### 桌位詳情功能
點擊用餐中的桌位會顯示詳細資訊彈窗：
- **桌位基本資訊**: 桌號、狀態、容納人數
- **用餐時間**: 自動計算從第一筆訂單開始的用餐時長
- **訂單列表**: 該桌所有未結帳的訂單詳情
- **總消費金額**: 該桌所有訂單的合計金額
- **快速操作**: 加點、結帳按鈕

## 🍽️ 視覺化點餐系統

### POS 風格界面
採用類似 iChef 的專業 POS 機設計：

#### 左側菜單區域
- **分類篩選標籤**: 全部、主食、飲料等快速篩選
- **格狀菜單**: 每個菜品獨立卡片顯示
- **即時反饋**: 點擊菜品有視覺動畫效果
- **清晰資訊**: 菜品名稱、價格、分類一目了然

#### 右側訂單區域
- **當前訂單**: 即時顯示已選擇的菜品
- **數量控制**: 簡單的 +/- 按鈕調整數量
- **項目管理**: 可隨時刪除不需要的項目
- **金額計算**: 自動計算小計和總額
- **快速清空**: 一鍵清除所有選擇

### 智能訂單處理
- **自動合併**: 重複選擇相同菜品會自動增加數量
- **即時更新**: 所有操作立即反映在訂單區域
- **錯誤防護**: 防止提交空白訂單

## ➕ 加點與結帳功能

### 加點流程
1. 點擊用餐中的桌位卡片
2. 在桌位詳情彈窗中點擊「加點」按鈕
3. 系統自動帶入桌號資訊
4. 使用相同的視覺化點餐界面選擇餐點
5. 確認後建立新的加點訂單

### 結帳流程
1. 在桌位詳情中查看該桌所有未結帳訂單
2. 確認總消費金額無誤
3. 點擊「結帳」按鈕
4. 系統確認後自動：
   - 將該桌所有訂單標記為「已完成」
   - 釋放桌位狀態為「空桌」
   - 更新今日營收統計

## 🍜 菜單管理

### 菜品資訊
- **新增菜品**: 設定名稱、價格、分類、描述
- **編輯功能**: 修改現有菜品的任何資訊
- **分類管理**: 支援主食、飲料等多種分類
- **批量操作**: 快速刪除不再供應的菜品

### 預設菜單
系統預設包含常見餐點：
- 招牌牛肉麵 ($180)
- 蔥爆牛肉 ($220)
- 宮保雞丁 ($160)
- 麻婆豆腐 ($120)
- 可樂 ($30)
- 熱茶 ($20)

## 📈 歷史記錄與分析

### 歷史記錄
- **訂單歷史**: 查看所有過往訂單記錄
- **時間篩選**: 按日期範圍篩選訂單
- **詳細資訊**: 每筆訂單的完整資訊
- **狀態追蹤**: 訂單狀態變更歷程

### 分析統計
- **營收分析**: 日、週、月營收趨勢
- **熱門菜品**: 銷量排行榜
- **桌位使用率**: 各桌位的使用情況
- **客流分析**: 用餐時段分布

## ⚙️ 系統設定

### Notion 整合設定
- **Integration Token**: 輸入 Notion API Token
- **Database ID**: 設定目標資料庫 ID
- **連線測試**: 自動驗證設定是否正確
- **同步狀態**: 顯示資料同步結果

### 系統偏好設定
- **介面語言**: 繁體中文
- **時區設定**: 台灣時區 (UTC+8)
- **自動儲存**: 設定自動儲存到 localStorage

## 🔄 Notion 同步機制

### 自動同步
所有透過系統建立的資料都會自動同步到 Notion：
- **訂單資料**: 包含完整訂單資訊
- **即時狀態**: 狀態變更即時更新
- **分離記錄**: 主訂單與加點訂單分別記錄
- **錯誤處理**: 同步失敗時的提示與重試

## 📱 響應式設計

### 多裝置支援
系統完全支援各種裝置：

#### 桌面電腦 (1024px+)
- **左右分欄**: 菜單與訂單並排顯示
- **完整功能**: 所有功能完整呈現

- **滑動支援**: 分類標籤支援水平滑動

- **大按鈕設計**: 適合手指觸控的按鈕尺寸
- **簡化介面**: 保留核心功能，優化操作流程
- **側邊欄選單**: 主選單收合為側邊欄

### 用戶體驗優化
- **載入動畫**: 平滑的頁面切換效果
- **視覺反饋**: 所有互動都有即時回饋
- **錯誤提示**: 清楚的錯誤訊息和解決建議
- **狀態保持**: 切換頁面時保持操作狀態

## 🎯 操作流程範例

### 完整點餐流程
1. **客人入座** → 點擊對應空桌
2. **開始點餐** → 選擇人數，進入視覺化點餐界面
3. **選擇菜品** → 使用分類篩選，點擊菜品卡片
4. **確認訂單** → 檢查右側訂單區域，確認送出
5. **處理訂單** → 在儀表板或歷史記錄中更新狀態
6. **客人加點** → 點擊桌位，使用加點功能
7. **完成結帳** → 在桌位詳情中結帳，釋放桌位

### 日常管理流程
1. **查看當日概況** → 儀表板檢視營收和訂單數
2. **管理進行中訂單** → 更新訂單狀態（製作中→已完成）
3. **桌位監控** → 查看各桌用餐時間和狀態
4. **菜單調整** → 新增特色菜品或調整價格
5. **資料分析** → 查看歷史記錄和統計分析
6. **系統維護** → 檢查 Notion 同步狀態
